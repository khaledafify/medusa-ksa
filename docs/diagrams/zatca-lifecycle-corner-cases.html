One source event, one legal document

Original sale, refund, return, and correction events each map to a stable source identity.

Signed documents are immutable

Corrections are new credit/debit notes that reference the original invoice, never edits.

Every document advances the chain

Invoices and notes all consume fresh ICV/PIH under the same serialized chain lock.

Reporting failure is operational state

Generation succeeds or fails independently from Fatoora reporting, which retries safely.
